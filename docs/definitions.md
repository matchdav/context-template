# Definitions

This file contains definitions of common terms used in this project.

## Core Concepts

*   **Context Repository**: A central repository for managing perspective, workflows, and automation across all active projects.
*   **Daily Digest**: A daily summary of active tickets, open pull requests, and recent team activity.
*   **Handoff**: The process of documenting and transferring work to another developer or to a future work session.
*   **Pickup**: The process of starting a new work session by reviewing the current context and relevant documents.

## Technical Terms

*   **MCP (Master Control Program)**: A term for the collection of services and automations that power the context ecosystem. Note: This term is being phased out in favor of a unified `context.json`.
*   **Service**: An external API or webhook that can be used by automations.
